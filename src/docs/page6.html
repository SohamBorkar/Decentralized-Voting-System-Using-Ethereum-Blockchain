Backend Technologies

The server-side components that power our decentralized voting platform

What Is a Backend and Why Is It Needed?

While the frontend is everything users see and interact with, the backend is the "behind-the-scenes" part of the application that:

  • Processes user requests and data
  • Communicates with databases and other services
  • Enforces business rules and security
  • Generates dynamic content for the frontend
Key Point: Even though our voting records are stored on the blockchain, we still need a traditional backend to handle user management, authentication, and to serve the web application itself.

Node.js: JavaScript on the Server

Node.js is a JavaScript runtime that allows developers to use JavaScript for server-side programming. Traditionally, different languages were used for frontend (JavaScript) and backend (like PHP, Java, or Python), but Node.js changed this by enabling JavaScript to run outside the browser.

Key benefits of Node.js in our application:

  • JavaScript Everywhere: Same language on frontend and backend for consistency
  • Non-Blocking I/O: Efficiently handles many concurrent users
  • NPM Ecosystem: Access to thousands of ready-to-use packages
  • Native JSON Support: Perfect for data exchange with modern web applications

Express: Web Framework for Node.js

Express is a minimal and flexible Node.js web application framework that provides a set of features for building web and mobile applications. It simplifies common web development tasks and helps organize application features.

Routing: Directs different user requests to the appropriate handling code

Middleware: Functions that process requests before they reach the final handler

API Creation: Simplified tools for building REST APIs

Static File Serving: Efficiently delivers HTML, CSS, and JavaScript files to browsers

Key Backend Components in Our Voting System

User Authentication System

Our backend implements a secure authentication system using:

  • JWT (JSON Web Tokens): Secure tokens for maintaining user sessions
  • Password Handling: Secure storage and verification of user credentials
  • Role-Based Access Control: Different permissions for voters and administrators
  • Session Management: Tracking logged-in users and their activities

FastAPI for Database Operations

We use FastAPI, a modern Python web framework, to create a high-performance API for database operations:

  • User registration and profile management
  • Credential verification during login
  • Storing personal information securely
  • Integration with the MySQL database

API Endpoints

Our backend exposes several API endpoints that the frontend uses to:

  • Register new users
  • Authenticate existing users
  • Retrieve candidate information
  • Submit votes to the blockchain
  • Fetch voting results

Blockchain Integration Layer

A critical part of our backend is the blockchain integration layer, which:

  • Connects to the Ethereum network
  • Interacts with our smart contracts
  • Processes blockchain transactions
  • Relays blockchain data to the frontend

Security Features

Our backend implements several security measures to protect the voting system:

CORS Protection: Prevents unauthorized websites from making requests to our API

Input Validation: Checks all user inputs to prevent malicious data

Rate Limiting: Prevents abuse by limiting the number of requests per time period

Secure Headers: Implements HTTP headers that enhance security

Error Handling: Properly manages errors without exposing sensitive information

These security measures, combined with the inherent security of blockchain technology, create a robust system that protects both the voting process and voter information.
